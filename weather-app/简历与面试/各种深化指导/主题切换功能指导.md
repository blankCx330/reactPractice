# 主题切换功能指导

> 本文档指导如何在 React + TypeScript + Tailwind CSS 项目中实现昼夜主题切换功能

---

## 一、功能概述

### 1.1 需求描述

实现一个主题切换按钮，用户可以手动切换「日间模式」和「夜间模式」，切换后：

- 全站颜色、背景、边框等样式自动适配
- 刷新页面后保持用户选择的主题
- 支持跟随系统主题（可选）

### 1.2 技术选型

| 方案                           | 优点                    | 缺点             | 推荐度     |
| ------------------------------ | ----------------------- | ---------------- | ---------- |
| CSS 变量 + class 切换          | 性能好、兼容性强        | 需要定义变量     | ⭐⭐⭐⭐⭐ |
| Zustand + Tailwind dark:       | 简单、Tailwind 原生支持 | 需要配合 Zustand | ⭐⭐⭐⭐⭐ |
| CSS-in-JS（styled-components） | 动态能力强              | 增加包体积       | ⭐⭐       |

**推荐方案**：Zustand + Tailwind `dark:` 前缀 + localStorage 持久化

---

## 二、实现方案

### 2.1 方案架构

```
用户点击切换按钮
       ↓
Zustand Store 更新 theme 状态
       ↓
<html> 标签添加/移除 class="dark"
       ↓
Tailwind dark: 前缀样式生效
       ↓
localStorage 持久化保存
```

### 2.2 核心代码

#### Step 1：创建主题 Store

```tsx
// src/stores/useThemeStore.ts
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

type Theme = 'light' | 'dark'

interface ThemeStore {
  theme: Theme
  toggleTheme: () => void
  setTheme: (theme: Theme) => void
}

export const useThemeStore = create<ThemeStore>()(
  persist(
    (set, get) => ({
      theme: 'light', // 默认日间模式

      toggleTheme: () => {
        const newTheme = get().theme === 'light' ? 'dark' : 'light'
        set({ theme: newTheme })
        updateDocumentTheme(newTheme)
      },

      setTheme: (theme: Theme) => {
        set({ theme })
        updateDocumentTheme(theme)
      },
    }),
    {
      name: 'theme-storage', // localStorage key
    }
  )
)

// 更新 <html> 标签的 class
function updateDocumentTheme(theme: Theme) {
  if (theme === 'dark') {
    document.documentElement.classList.add('dark')
  } else {
    document.documentElement.classList.remove('dark')
  }
}
```

#### Step 2：应用初始化时同步主题

```tsx
// src/App.tsx
import { useEffect } from 'react'
import { useThemeStore } from './stores/useThemeStore'

function App() {
  const theme = useThemeStore(state => state.theme)

  // 初始化时同步主题到 DOM
  useEffect(() => {
    if (theme === 'dark') {
      document.documentElement.classList.add('dark')
    } else {
      document.documentElement.classList.remove('dark')
    }
  }, [])

  return (
    // ...
  )
}
```

## 四、高级功能

### 4.1 跟随系统主题

```tsx
// src/stores/useThemeStore.ts
type Theme = 'light' | 'dark' | 'system'

interface ThemeStore {
  theme: Theme
  toggleTheme: () => void
  setTheme: (theme: Theme) => void
}

export const useThemeStore = create<ThemeStore>()(
  persist(
    (set, get) => ({
      theme: 'system',

      toggleTheme: () => {
        const themes: Theme[] = ['light', 'dark', 'system']
        const currentIndex = themes.indexOf(get().theme)
        const newTheme = themes[(currentIndex + 1) % themes.length]
        set({ theme: newTheme })
        applyTheme(newTheme)
      },

      setTheme: (theme: Theme) => {
        set({ theme })
        applyTheme(theme)
      },
    }),
    {
      name: 'theme-storage',
    }
  )
)

function applyTheme(theme: Theme) {
  const isDark =
    theme === 'dark' ||
    (theme === 'system' && window.matchMedia('(prefers-color-scheme: dark)').matches)

  if (isDark) {
    document.documentElement.classList.add('dark')
  } else {
    document.documentElement.classList.remove('dark')
  }
}

// 监听系统主题变化
function useSystemThemeListener() {
  const setTheme = useThemeStore(state => state.setTheme)
  const currentTheme = useThemeStore(state => state.theme)

  useEffect(() => {
    if (currentTheme !== 'system') return

    const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)')

    const handleChange = (e: MediaQueryListEvent) => {
      applyTheme('system')
    }

    mediaQuery.addEventListener('change', handleChange)
    return () => mediaQuery.removeEventListener('change', handleChange)
  }, [currentTheme])
}
```

### 4.2 过渡动画

```tsx

```

### 4.3 防止闪烁（SSR/初始加载）

```html
<!-- index.html 的 <head> 中添加 -->
<script>
  // 在 DOM 加载前执行，防止白屏闪烁
  ;(function () {
    const theme = localStorage.getItem('theme-storage')
    if (theme) {
      const parsed = JSON.parse(theme)
      if (parsed.state?.theme === 'dark') {
        document.documentElement.classList.add('dark')
      }
    }
  })()
</script>
```

## 六、面试考点

### 6.1 常见问题

**Q1：为什么使用 `darkMode: 'class'` 而不是 `darkMode: 'media'`？**

> `class` 模式允许用户手动控制，`media` 模式只能跟随系统。实际产品中用户需要手动切换的能力，所以用 `class`。

**Q2：为什么用 Zustand 而不是 React Context？**

> Zustand 自带持久化中间件，无需手动处理 localStorage；性能更好，只有订阅该状态的组件才会重渲染；代码更简洁。

**Q3：为什么在 `<html>` 上加 `dark` 类而不是 `<body>`？**

> Tailwind 官方推荐在 `<html>` 上添加，这样可以覆盖全局样式，包括 `body` 和其他根元素。而且 Tailwind 的 `dark:` 前缀默认检查 `html` 上的 class。

**Q4：如何解决页面加载时的闪烁问题？**

> 在 HTML 的 `<head>` 中添加内联脚本，在 DOM 渲染前读取 localStorage 并设置主题类，避免白屏闪烁。

### 6.2 代码解读

```tsx
// 这段代码的作用是什么？
useEffect(() => {
  if (theme === 'dark') {
    document.documentElement.classList.add('dark')
  } else {
    document.documentElement.classList.remove('dark')
  }
}, [])
```

> 答案：组件初始化时，从 localStorage 恢复的主题状态同步到 DOM。因为 Zustand 的 persist 只更新 store，不会自动操作 DOM，所以需要在初始化时手动同步。

## 八、总结

### 核心要点

1. **状态管理**：使用 Zustand + persist 中间件管理主题状态
2. **样式控制**：Tailwind 的 `dark:` 前缀 + `darkMode: 'class'` 配置
3. **DOM 操作**：在 `<html>` 上添加/移除 `dark` 类
4. **持久化**：localStorage 保存用户选择
5. **初始化同步**：useEffect 中同步 store 状态到 DOM

### 实现口诀

> **Store 管状态，Toggle 换主题，dark: 写样式，persist 记选择**
