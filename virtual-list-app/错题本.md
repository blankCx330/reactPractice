### 问题 #1: useDebouncedValue Hook 初始值设置错误

**首次记录**: 2026-03-23 00:02
**最近记录**: 2026-03-23 00:02
**记录次数**: 1

**原始问题**:
> 我这里初始值设置为''有什么问题？

**相关代码**:
```ts
import { useState, useEffect } from "react";

export default function useDebouncedValue(data:string, time:number){
    const [debounced, setDebounce] = useState('')  // 问题：初始值是空字符串

    useEffect(()=>{
        const timer = setTimeout(()=>setDebounce(data), time)
        return () => {
            clearTimeout(timer)
        }
    },[data, time])

    return debounced
}
```

**问题分析**:
初始值设为 `''` 导致首次渲染返回错误值。防抖（debounce）的核心是：快速变化时只取最后一次值，但第一次应该立即返回。如果初始值是 `''`，用户传入 `'张三'`，首次渲染会返回 `''`，500ms 后才变成 `'张三'`——这不是"防抖"，而是"延迟"。

**正确做法**:
```ts
import { useState, useEffect } from "react";

export default function useDebouncedValue<T>(data: T, time: number): T {
    const [debounced, setDebounced] = useState<T>(data)  // 用 data 作为初始值

    useEffect(() => {
        const timer = setTimeout(() => setDebounced(data), time)
        return () => clearTimeout(timer)
    }, [data, time])

    return debounced
}
```

**改进点**:
1. 初始值用 `data` —— 首次渲染就返回正确值
2. 加了泛型 `<T>` —— 不再局限于 `string`，可以处理任何类型

**学习要点**: ⭐ 防抖 Hook 的初始值应该是传入的 `data`，不是硬编码的默认值。第一次就应该正确，后面才防抖。

### 问题 #2: Flexbox justify-center 导致列表被遮蔽

**首次记录**: 2026-03-23 13:30
**最近记录**: 2026-03-23 13:30
**记录次数**: 1

**原始问题**:
> 为什么我的item会被搜索栏遮蔽一部分？

**相关代码**:
```tsx
// App.tsx - 父容器
<div className='
      border-2 w-1/3 h-full 
      flex flex-col justify-center items-center  // 问题：justify-center
    '
>
  <SearchBar />
  <ArticleList />
</div>

// ArticleList/index.tsx
<div className="
        bg-green-100
        w-full
        flex flex-col justify-center items-center  // 问题：justify-center
        overflow-auto
    "
>
```

**问题分析**:
父容器使用 `justify-center` 导致子元素在垂直方向居中排列，而不是从顶部开始。ArticleList 从屏幕中间开始渲染，滚动时顶部内容被 SearchBar 遮住。

**正确做法**:
```tsx
// App.tsx - 父容器
<div className='
      border-2 w-1/3 h-full 
      flex flex-col          // 移除 justify-center 和 items-center
    '
>
  <SearchBar />
  <ArticleList />
</div>

// ArticleList/index.tsx
<div className="
        bg-green-100
        w-full
        flex flex-col justify-start items-center  // justify-start：从顶部开始
        overflow-auto
        flex-1                                    // flex-1：填充剩余空间
    "
>
```

**关键修改**:
1. 移除父容器的 `justify-center` —— 内容从顶部开始排列
2. ArticleList 添加 `flex-1` —— 自动填充 SearchBar 之外的所有剩余空间
3. ArticleList 使用 `justify-start` —— 列表内容从顶部开始

**学习要点**: ⭐ `flex-1` + `overflow-auto` = 可滚动区域的黄金组合。`flex-1` 让元素占据剩余空间，`overflow-auto` 让它可以滚动。父容器不要用 `justify-center`，否则内容会从中间开始。

### 问题 #3: react-window v2 类型定义不匹配

**首次记录**: 2026-03-23 15:00
**最近记录**: 2026-03-23 15:00
**记录次数**: 1

**原始问题**:
> 模块"react-window"没有导出的成员"FixedSizeList"。这是为什么？

**相关代码**:
```tsx
import {FixedSizeList as List} from 'react-window'  // 报错！
```

**问题分析**:
项目安装了 **react-window v2.2.7**，但类型定义 `@types/react-window` 是 **v1.8.8**。v2 进行了重大 API 变更：
- v1.x 导出：`FixedSizeList`, `VariableSizeList`
- v2.x 导出：`List`, `Grid`（统一 API）

TypeScript 使用旧的类型定义检查代码，不认识 v2 的导出。

**正确做法**:
```bash
# 方案一：降级到 v1（推荐，稳定）
npm uninstall react-window
npm install react-window@1.8.10

# 方案二：升级类型定义（如果官方已更新）
npm install @types/react-window@latest
```

```tsx
// v1 写法（稳定，文档丰富）
import { FixedSizeList as List } from 'react-window';

// v2 写法（需要类型定义支持）
import { List } from 'react-window';
```

**学习要点**: ⭐ 库版本和类型定义版本必须匹配！当 TypeScript 报错"模块没有导出成员"时，先检查 `package.json` 中的库版本和 `@types/*` 版本是否一致。

### 问题 #4: react-window v2 属性名变化

**首次记录**: 2026-03-23 16:00
**最近记录**: 2026-03-23 16:00
**记录次数**: 1

**原始问题**:
> 类型上不存在属性"height"。这是怎么回事？

**相关代码**:
```tsx
import { List } from 'react-window';

<List
    height={400}        // ❌ v2 不存在这个属性
    itemCount={arr.length}
    itemSize={50}
    width="100%"
>
</List>
```

**问题分析**:
react-window v2 进行了破坏性 API 更新，属性名全部改变：

| v1 API | v2 API |
|--------|--------|
| `height` | 通过 `style` 或父容器设置 |
| `itemCount` | `rowCount` |
| `itemSize` | `rowHeight` |
| `width` | 通过 `style` 或父容器设置 |
| `children`（渲染函数） | `rowComponent` |

**正确做法**:
```bash
# 推荐：降级到 v1（文档丰富、稳定）
npm uninstall react-window
npm install react-window@1.8.10
```

```tsx
// v1 写法（推荐学习使用）
import { FixedSizeList as List } from 'react-window';

<List
    height={400}
    itemCount={arr.length}
    itemSize={50}
    width="100%"
>
    {({ index, style }) => (
        <div style={style}>{items[index]}</div>
    )}
</List>
```

**学习要点**: ⭐ react-window v1 和 v2 的 API 完全不兼容！v1 用 `height/itemCount/itemSize`，v2 用 `rowCount/rowHeight/rowComponent`。学习项目建议用 v1，文档多、教程丰富。

### 问题 #6: react-window v2 rowProps 解构错误

**首次记录**: 2026-03-23 20:00
**最近记录**: 2026-03-23 20:00
**记录次数**: 1

**原始问题**:
> 类型"RowComponentProps<RowData>"上不存在属性"rowProps"。

**相关代码**:
```tsx
// ❌ 错误写法
interface RowData { items: string[]; }

const RowComponent = ({ index, style, rowProps }: RowComponentProps<RowData>) => {
    const item = rowProps.items[index];  // rowProps 不存在！
}
```

**问题分析**:
react-window v2 的 `rowProps` 属性会**直接展开**到组件 props 中，不是作为嵌套对象！

```typescript
// 实际类型定义
rowComponent: (props: {
    index: number;
    style: CSSProperties;
    ariaAttributes: { ... };
} & RowProps) => ReactElement;  // ← RowProps 直接展开合并！
```

**正确做法**:
```tsx
// ✅ 正确：直接解构 rowProps 中的属性
interface RowData { items: string[]; }

const RowComponent = ({ index, style, items }: RowComponentProps<RowData>) => {
    const item = items[index];  // 直接访问！
    return <div style={style}>{item}</div>;
}

// 使用时
<List
    rowComponent={RowComponent}
    rowProps={{ items }}  // 传入 { items: [...] }
/>
```

**学习要点**: ⭐ `rowProps` 的属性会"展开"到组件 props 中！传入 `rowProps={{ items, onClick }}`，接收时直接解构 `({ index, style, items, onClick })`，**不是** `({ rowProps })`。

### 问题 #7: 函数组件 TypeScript 类型定义错误

**首次记录**: 2026-03-23 23:00
**最近记录**: 2026-03-23 23:00
**记录次数**: 1

**原始问题**:
> 为什么会报错？又或者说我应该如何规范的在 List 中使用我的组件？

**相关代码**:
```tsx
// ❌ 多处错误
export default function({num, style?}:{num:string, style:string}){
    // 1. 函数没有名字
    // 2. style? 位置错误（应该在类型定义里）
    // 3. style 类型错误（应该是 CSSProperties，不是 string）
    return <div style={style}>...</div>
}
```

**问题分析**:
1. **匿名函数**：组件应该有名字，便于调试和类型推断
2. **可选参数语法错误**：`style?` 应该写在类型定义中，即 `style?: Type`
3. **style 类型错误**：React 的 `style` 是 `CSSProperties` 对象，不是 `string`

**正确做法**:
```tsx
import type { CSSProperties } from 'react';

// ✅ 定义 props 类型
interface ArticleItemProps {
  num: string;
  style?: CSSProperties;  // 可选的样式对象
}

// ✅ 有名字的函数组件
export default function ArticleItem({ num, style }: ArticleItemProps) {
  return (
    <div style={style} className="...">
      <h1>{num}这是仓库名字</h1>
      {/* ... */}
    </div>
  );
}

// ✅ 在 List 中使用
interface RowProps { items: string[]; }

const RowComponent = ({ index, style, items }: RowComponentProps<RowProps>) => {
  return <ArticleItem num={items[index]} style={style} />;
};
```

**学习要点**: ⭐ 组件应有名字和正确的类型定义。`style` 类型是 `CSSProperties`，可选参数写作 `style?: Type`。在 List 中使用时，RowComponent 解构数据后传给你的组件。

### 问题 #8: 手写虚拟列表多个关键错误

**首次记录**: 2026-03-24 10:30
**最近记录**: 2026-03-24 11:00
**记录次数**: 2

**原始问题**:
> 检查一下这个手撕虚拟列表有什么不足，要指导我改正

**历史问题**:
> - [2026-03-24 11:00] 现在批改一下这个手撕虚拟列表有什么不足

**修复进度**:
- ✅ onScroll 放在外层容器
- ✅ 内部容器添加 position: relative
- ✅ 变量名改为 scrollTop
- ✅ 循环条件用 <=
- ✅ 参数名改为 overscan（但有拼写错误：owerscan）

**仍存在的问题**:
1. 🔴 **致命 bug**：`totalHeight = itemHeight * itemHeight` 应该是 `itemHeight * itemCount`
2. 🟡 **拼写错误**：`owerscan` → `overscan`
3. 🟡 **缺少 key**：循环渲染仍缺少 key 属性
4. 🟢 **性能优化**：可用 useMemo/useCallback 优化

**相关代码（第二轮）**:
```tsx
const totalHeight = itemHeight * itemHeight  // ❌ 致命 bug！应该是 itemCount
owerscan?: number,  // ❌ 拼写错误，应该是 overscan
```

**问题分析**:
发现 6 个问题：
1. **onScroll 放错位置**：应该放在外层滚动容器，而不是内部撑高元素
2. **缺少 key 属性**：循环渲染没有 key，React 无法正确识别列表项
3. **内部容器缺少 position: relative**：子元素 absolute 定位需要父元素相对定位
4. **endIndex 循环条件错误**：`index < endIndex` 应该是 `index <= endIndex`
5. **变量命名不清晰**：`scroll` 应该是 `scrollTop`，`advance` 建议改为 `overscan`
6. **性能优化空间**：可使用 useMemo 缓存 items，useCallback 缓存滚动处理函数

**正确做法**:
```tsx
import React, { useState, useMemo, useCallback } from "react";

interface FixedSizeListProps {
    height: number;
    itemHeight: number;
    itemCount: number;
    overscan?: number;  // 业界通用术语
    children: (props: { index: number; style: React.CSSProperties }) => React.ReactNode;
}

export function FixedSizeList({ 
    height, itemCount, itemHeight, overscan = 3, children 
}: FixedSizeListProps) {
    
    const [scrollTop, setScrollTop] = useState(0);

    const { startIndex, endIndex } = useMemo(() => {
        const startIndex = Math.max(0, Math.floor(scrollTop / itemHeight) - overscan);
        const visibleCount = Math.ceil(height / itemHeight);
        const endIndex = Math.min(itemCount - 1, startIndex + visibleCount + overscan);
        return { startIndex, endIndex };
    }, [scrollTop, itemHeight, itemCount, overscan, height]);

    const totalHeight = itemHeight * itemCount;

    const handleScroll = useCallback((e: React.UIEvent<HTMLDivElement>) => {
        setScrollTop(e.currentTarget.scrollTop);
    }, []);

    const items = useMemo(() => {
        const result = [];
        for (let index = startIndex; index <= endIndex; index++) {
            result.push(
                <div key={index}>
                    {children({
                        index,
                        style: {
                            height: itemHeight,
                            position: 'absolute',
                            width: '100%',
                            top: index * itemHeight
                        }
                    })}
                </div>
            );
        }
        return result;
    }, [startIndex, endIndex, children, itemHeight]);

    return (
        <div style={{ height, overflow: 'auto' }} onScroll={handleScroll}>
            <div style={{ height: totalHeight, position: 'relative' }}>
                {items}
            </div>
        </div>
    );
}
```

**学习要点**: ⭐ 
- `onScroll` 放在 `overflow: auto` 的滚动容器上
- 列表渲染必须有 `key`
- 子元素 `absolute` 时父元素要 `relative`
- 用 `overscan`（预渲染）代替 `advance`，这是业界通用术语

### 问题 #1: .ts 与 .tsx 文件扩展名的区别

**首次记录**: 2026-03-23 00:00
**最近记录**: 2026-03-23 00:01
**记录次数**: 2

**原始问题**:
> .tsx 和 .ts 有什么区别？为什么hook用.ts而组件用.tsx 这个是严格规定吗？

**历史问题**:
> - [2026-03-23 00:01] 也就是说我全部都用tsx也没什么影响对吗？

**知识点**: TypeScript / 文件扩展名 / JSX

**核心解答**:
`.tsx` = `.ts` + JSX 语法支持。有 JSX 就用 `.tsx`，没有就用 `.ts`。

**深入理解**:
JSX 语法中有 `< >` 符号，这在 TypeScript 里也用于泛型。如果是 `.ts` 文件，编译器会把 `< >` 当作泛型解析，遇到 JSX 就会报错。

**追问解答**:
技术上可以全部用 `.tsx`，代码能正常运行。但存在三个问题：
1. **可读性差** - 看到 `.tsx` 会以为是组件，打开发现是纯逻辑函数，浪费认知成本
2. **IDE 提示干扰** - VS Code 在 `.tsx` 文件中会提供 JSX 相关的自动补全，在纯逻辑文件中这些提示是噪音
3. **团队规范** - 业界约定 `.ts` = 纯逻辑，`.tsx` = 包含 JSX，违反会让队友困惑

**代码示例**:
```tsx
// .tsx 文件 - 组件（包含 JSX）
const Button = () => <button>Click</button>;

// .ts 文件 - Hook（纯逻辑，无 JSX）
const useCounter = () => {
  const [count, setCount] = useState(0);
  return { count, increment: () => setCount(c => c + 1) };
};
```

**学习要点**: ⭐ 有 JSX 就用 `.tsx`，没有就用 `.ts`。能做不代表应该做——按规范区分是为了让代码"会说话"。

### 问题 #2: index.tsx 文件命名约定

**首次记录**: 2026-03-23 00:03
**最近记录**: 2026-03-23 00:03
**记录次数**: 1

**原始问题**:
> index.tsx 是什么？是 import ArticleList from ... 吗？那为啥不直接以引用的组件的名字来命名文件?index是有什么别的意思吗？还是说这是一种默契?

**知识点**: 模块导入机制 / 文件命名规范 / index 入口

**核心解答**:
`index.tsx` 是模块入口文件。导入文件夹时，会自动查找该文件夹下的 `index` 文件，可以省略文件名。这不是规定，是约定。

**深入理解**:
`index` 来自"索引"概念，就像书的目录。当导入一个文件夹路径时（如 `./components/Button`），模块系统会自动查找 `Button/index.tsx`。

**好处**:
1. **重命名方便** —— 只改文件夹名，不用改导入路径
2. **组织复杂组件** —— 可以在文件夹内放子组件、样式、Hook 等，只通过 index 对外暴露
3. **导入路径简洁** —— `import ArticleList from './ArticleList'` 比 `import ArticleList from './ArticleList/ArticleList'` 更清晰

**代码示例**:
```
components/
└── ArticleList/
    ├── index.tsx         ← 对外暴露的主组件
    ├── ArticleItem.tsx   ← 内部子组件
    ├── useArticle.ts     ← 内部 Hook
    └── styles.css        ← 内部样式
```

```tsx
// 导入时简洁
import ArticleList from './components/ArticleList';
// 等同于
import ArticleList from './components/ArticleList/index';
```

**学习要点**: ⭐ `index.tsx` = 文件夹的默认入口。导入文件夹时自动找它，可以省略文件名。这是一种约定，让导入路径更简洁，方便组织复杂组件。

### 问题 #3: Tailwind 设置全屏背景

**首次记录**: 2026-03-23 00:04
**最近记录**: 2026-03-23 00:04
**记录次数**: 1

**原始问题**:
> 如何使用 Tailwind 设置一个全屏的背景？应该在我的项目中的哪个组件设置？

**知识点**: Tailwind CSS / 全屏布局 / 背景样式

**核心解答**:
全屏背景的关键是 `min-h-screen`（最小高度 = 视口高度）。推荐在根组件 `App.tsx` 设置，用 `div` 包裹所有内容。

**深入理解**:
- `min-h-screen`：最小高度 = 100vh，内容少时撑满屏幕，内容多时自动扩展
- `h-screen`：固定高度 = 100vh，内容多会溢出，不推荐
- `w-full`：宽度 = 100%

**代码示例**:
```tsx
// src/App.tsx
function App() {
  return (
    <div className="min-h-screen bg-gradient-to-b from-blue-500 to-purple-600">
      {/* 所有内容 */}
    </div>
  )
}

// 常用背景样式：
// 纯色：bg-gray-100
// 渐变：bg-gradient-to-r from-cyan-500 to-blue-500
// 图片：bg-cover bg-center + style={{ backgroundImage: 'url(/bg.jpg)' }}
```

**学习要点**: ⭐ 全屏背景 = `min-h-screen` + `bg-*`。设置在根组件 `App.tsx`，用 `div` 包裹所有内容。用 `min-h-screen` 而非 `h-screen`，避免内容溢出。

### 问题 #4: React 组件 Props 传参

**首次记录**: 2026-03-23 15:30
**最近记录**: 2026-03-23 15:30
**记录次数**: 1

**原始问题**:
> `<ArticleItem num />` 我应该如何传参？

**相关代码**:
```tsx
// 错误的组件定义
export default function(num: any) {  // ❌ 参数应该是 props 对象
    return <h1>{num}</h1>
}
```

**知识点**: React 组件 / Props / 参数传递

**核心解答**:
React 组件通过 **props 对象** 接收外部传入的数据。组件函数的第一个参数就是 props 对象，需要解构才能使用具体属性。

**深入理解**:
```tsx
// ✅ 正确的组件定义
interface ArticleItemProps {
    num: number;
    title?: string;  // 可选属性
}

export default function ArticleItem({ num, title = "默认标题" }: ArticleItemProps) {
    return <h1>{num}: {title}</h1>;
}
```

**代码示例**:
```tsx
// 使用组件时传参
<ArticleItem num={1} />              // 传递数字
<ArticleItem num={1} title="Hello" /> // 传递多个参数
<ArticleItem num={variable} />        // 传递变量

// ⚠️ 注意区别
<ArticleItem num />      // 等于 num={true}，传递布尔值
<ArticleItem num={1} />  // 传递数字 1
```

**学习要点**: ⭐ Props 是对象，传参用花括号。定义组件用 `({ name }: { name: string })`，使用组件用 `<Comp name="值" />`。记住：`<Comp name />` 等于 `name={true}`，不是 `name="name"`！

### 问题 #5: react-window List 组件使用

**首次记录**: 2026-03-23 21:00
**最近记录**: 2026-03-23 21:00
**记录次数**: 1

**原始问题**:
> 我应该如何使用 List，又应该如何正确的导入我的数据来自定义我的 List？

**知识点**: react-window / 虚拟列表 / 组件配置

**核心解答**:
List 使用三步走：定义 RowProps 类型 → 编写行组件（直接解构数据）→ 配置 List 组件（rowComponent、rowCount、rowHeight、rowProps）。

**深入理解**:
```tsx
// 1. 定义数据类型
interface RowProps {
  items: string[];  // 或 repos: Repo[], articles: Article[]
}

// 2. 行组件 - ⚠️ 直接解构 rowProps 中的属性
const RowComponent = ({ index, style, items }: RowComponentProps<RowProps>) => {
  // index: 当前行索引
  // style: 必须应用到根元素
  // items: 来自 rowProps={{ items }}
  return <div style={style}>{items[index]}</div>;
};

// 3. 配置 List
<List
  rowComponent={RowComponent}   // 行组件
  rowCount={items.length}       // 总数
  rowHeight={50}                // 行高
  rowProps={{ items }}          // 数据（属性会展开到行组件 props）
  style={{ height: '100%', width: '100%' }}
/>
```

**关键点**:
1. `rowProps` 的属性会**直接展开**到行组件 props 中
2. 父容器必须有**明确高度**
3. `style` 必须应用到行组件根元素

**学习要点**: ⭐ List 使用三步走：定义 RowProps → 行组件解构数据 `({ index, style, items })` → `rowProps={{ items }}` 传入。记住：rowProps 属性会"展开"！

### 问题 #6: RowComponent 工作原理

**首次记录**: 2026-03-23 22:00
**最近记录**: 2026-03-23 22:00
**记录次数**: 1

**原始问题**:
> RowComponent 的命名是一定的吗？ index,style 是怎么来的？这个组件是会自行解析数组还是因为传入 List 后解析的？以及这个 style 有什么必要吗？

**知识点**: react-window / RowComponent / 虚拟列表原理

**核心解答**:
RowComponent 是"单行渲染器"，命名随意。`index` 和 `style` 由 List 自动传入。List 负责遍历数组，RowComponent 只渲染单行。`style` 控制位置（必需），`className` 控制外观（可选）。

**深入理解**:
```tsx
// RowComponent 命名随意
const Row = ({ index, style, items }) => ...      // ✅
const ItemRenderer = ({ index, style, items }) => ...  // ✅

// List 内部工作原理（伪代码）
for (let i = startIndex; i <= endIndex; i++) {
  RowComponent({
    index: i,           // List 传入当前行索引
    style: {            // List 计算位置样式
      position: 'absolute',
      top: i * rowHeight,
    },
    ...rowProps         // 你的数据
  })
}

// style 的作用：控制行的位置
<div style={style} className="my-style">
  {/*  ↑ 位置（必须）    ↑ 外观（可选） */}
</div>
```

**关键区别**:
| 属性 | 来源 | 作用 |
|------|------|------|
| `index` | List 计算 | 当前行索引 |
| `style` | List 计算 | 控制位置（必须应用） |
| `className` | 你定义 | 控制外观（可选） |
| `items` 等 | `rowProps` | 你的数据 |

**学习要点**: ⭐ RowComponent 是"单行渲染器"：`index`、`style` 由 List 传入；`style` 控制位置必须应用；List 负责遍历，RowComponent 只渲染当前行。
