#### 任务 1.1：手写定高虚拟列表（核心）

**目标**：能手写一个简单的虚拟列表，理解核心原理

**新建文件**：`src/components/VirtualList/FixedSizeList.tsx`

```tsx
/**
 * 手写虚拟列表 - 定高版本
 * 
 * 核心原理：
 * 1. 只渲染可视区域内的元素（O(1) DOM节点）
 * 2. 使用 transform/padding 撑开滚动区域
 * 3. 监听滚动事件，动态计算 startIndex 和 endIndex
 */

import { useRef, useState, useEffect, useMemo } from 'react'

interface FixedSizeListProps {
  height: number           // 可视区域高度
  itemCount: number        // 总数据量
  itemHeight: number       // 每项高度（固定）
  overscan?: number        // 缓冲区数量，防止滚动白屏
  children: (props: { index: number; style: React.CSSProperties }) => React.ReactNode
}

export default function FixedSizeList({
  height,
  itemCount,
  itemHeight,
  overscan = 3,
  children,
}: FixedSizeListProps) {
  const containerRef = useRef<HTMLDivElement>(null)
  const [scrollTop, setScrollTop] = useState(0)

  // 处理滚动事件（可以加节流优化）
  const handleScroll = (e: React.UIEvent<HTMLDivElement>) => {
    setScrollTop(e.currentTarget.scrollTop)
  }

  // 计算可视区域的起始和结束索引
  const { startIndex, endIndex, visibleCount } = useMemo(() => {
    const start = Math.floor(scrollTop / itemHeight)
    const visible = Math.ceil(height / itemHeight)
    const end = Math.min(start + visible + overscan, itemCount - 1)
    
    return {
      startIndex: Math.max(0, start - overscan),
      endIndex: end,
      visibleCount: visible,
    }
  }, [scrollTop, itemHeight, height, itemCount, overscan])

  // 总高度（撑开滚动区域）
  const totalHeight = itemCount * itemHeight

  // 偏移量（让渲染的元素出现在正确位置）
  const offsetY = startIndex * itemHeight

  // 实际渲染的元素
  const items = []
  for (let i = startIndex; i <= endIndex; i++) {
    items.push(
      children({
        index: i,
        style: {
          position: 'absolute',
          top: i * itemHeight,
          height: itemHeight,
          width: '100%',
        },
      })
    )
  }

  return (
    <div
      ref={containerRef}
      style={{ height, overflow: 'auto', position: 'relative' }}
      onScroll={handleScroll}
    >
      {/* 撑开滚动区域 */}
      <div style={{ height: totalHeight, position: 'relative' }}>
        {/* 渲染可视区域的元素 */}
        <div style={{ transform: `translateY(${offsetY}px)` }}>
          {items}
        </div>
      </div>
    </div>
  )
}
```

**面试要点**：

| 问题 | 答案 |
|------|------|
| 为什么需要虚拟列表？ | DOM 节点过多导致重绘重排开销大，10000个节点会严重卡顿 |
| 核心思想是什么？ | 只渲染可视区域内的元素，DOM节点从 O(n) 降为 O(1) |
| startIndex 如何计算？ | `Math.floor(scrollTop / itemHeight)` |
| 缓冲区(overscan)的作用？ | 防止快速滚动时出现白屏，提前渲染可视区域外的元素 |
| 为什么用 absolute 定位？ | 让每个元素精确定位到正确位置，不依赖文档流 |

import FixedSizeList from '../components/VirtualList/FixedSizeList'

          普通渲染
        </button>
        <button
          onClick={() => setRenderMode('virtual')}
          className={`px-4 py-2 rounded ${
            renderMode === 'virtual' ? 'bg-blue-500 text-white' : 'bg-gray-200'
          }`}
        >
          虚拟列表
        </button>
      </div>

      <div className="border rounded-lg">
        {renderMode === 'normal' ? <NormalList /> : <VirtualList />}
      </div>

      <div className="mt-4 text-gray-600">
        <p>切换渲染模式，对比滚动流畅度</p>
        <p>普通渲染：DOM 节点 = 数据量 = 10,000</p>
        <p>虚拟列表：DOM 节点 ≈ 可视区域 ≈ 10</p>

  )

import { useQuery } from "@tanstack/react-query";
import type { GitHubSearchResponse } from "../../types/article";

interface UseArticleListParams {
  searchQuery?: string;
  language?: string;
}

export const useArticleList = ({ searchQuery = '', language = '' }: UseArticleListParams = {}) => {
  // 构建搜索查询
  const buildQuery = () => {
    let q = 'stars:>100'; // 基础条件：star数大于100
    
    if (searchQuery) {
      q += ` ${searchQuery}`;
    }
    
    if (language) {
      q += ` language:${language}`;
    }
    
    return q;
  };

  const query = buildQuery();
  
  const url = `https://api.github.com/search/repositories?q=${encodeURIComponent(query)}&sort=stars&order=desc&per_page=30`;

  return useQuery<GitHubSearchResponse>({
    queryKey: ['repositories', searchQuery, language],
    queryFn: async () => {
      const response = await fetch(url);
      if (!response.ok) {
        throw new Error(`HTTP error! status: ${response.status}`);
      }
      return response.json();
    },
    staleTime: 5 * 60 * 1000, // 5分钟内不重新请求
    gcTime: 10 * 60 * 1000,   // 10分钟后清除缓存
  });
};

import { useState, useCallback } from 'react'
import ArticleItem from "./ArticleItem"
import { useArticleList } from "./useArticleList"
import type { GitHubRepository } from "../../types/article"
import { List, type RowComponentProps } from "react-window"
import SearchBar from "../SearchBar"

type RowData = {
  items: GitHubRepository[]

      name: item.full_name,

      <div style={{ ...style, paddingBottom: 8 }}>
        <ArticleItem data={articleData} />
      </div>
    )

  if (isLoading) {
    return (
      <div className="flex flex-col h-full">
        <SearchBar onSearch={handleSearch} onLanguageChange={handleLanguageChange} />
        <div className="flex-1 flex items-center justify-center">
          <div className="text-gray-500">加载中...</div>
        </div>
      </div>
    )
  }

  // 错误状态
  if (isError) {
    return (
      <div className="flex flex-col h-full">

          <div className="text-red-500">

          </div>
          <button
            onClick={() => refetch()}
            className="px-4 py-2 bg-blue-500 text-white rounded-lg hover:bg-blue-600"
          >
            重试
          </button>
        </div>
      </div>
    )
  }

  // 空状态
  if (!data?.items?.length) {
    return (
      <div className="flex flex-col h-full">
        <SearchBar onSearch={handleSearch} onLanguageChange={handleLanguageChange} />
        <div className="flex-1 flex items-center justify-center">
          <div className="text-gray-500">没有找到相关仓库</div>
        </div>
      </div>
    )
  }

  return (
    <div className="flex flex-col h-full bg-gray-50">
      <SearchBar onSearch={handleSearch} onLanguageChange={handleLanguageChange} />
      <div className="flex-1">
        <List
          rowComponent={RowComponent}
          rowCount={data.items.length}
          rowHeight={80}
          rowProps={{ items: data.items }}
          style={{ height: '100%', width: '100%' }}
        />
      </div>
    </div>
  )
}

### Day 3：不定高虚拟列表（可选，3-4小时）

#### 任务 3.1：理解不定高的挑战

**问题**：
- 定高：每个 item 高度已知，可以直接计算位置
- 不定高：每个 item 高度不确定，需要预估 + 实测

**解决方案**：
1. **预估高度**：先给一个默认高度
2. **渲染后测量**：渲染后获取真实高度
3. **缓存高度**：记录已测量的高度
4. **位置修正**：根据真实高度重新计算位置

#### 任务 3.2：手写不定高虚拟列表

**新建文件**：`src/components/VirtualList/VariableSizeList.tsx`

```tsx
/**
 * 手写虚拟列表 - 不定高版本
 * 
 * 核心难点：
 * 1. 无法预先知道每个 item 的高度
 * 2. 需要预估 + 实测修正
 * 3. 需要缓存已测量的高度
 */

import { useRef, useState, useEffect, useMemo, useCallback } from 'react'

interface VariableSizeListProps {
  height: number
  itemCount: number
  estimatedItemHeight: number  // 预估高度
  overscan?: number
  children: (props: { index: number; style: React.CSSProperties }) => React.ReactNode
}

export default function VariableSizeList({
  height,
  itemCount,
  estimatedItemHeight,
  overscan = 3,
  children,
}: VariableSizeListProps) {
  const containerRef = useRef<HTMLDivElement>(null)
  const [scrollTop, setScrollTop] = useState(0)
  
  // 高度缓存：记录每个 item 的真实高度
  const heightCache = useRef<number[]>([])
  // item 位置缓存：记录每个 item 的 top 位置
  const positionCache = useRef<number[]>([])

  // 获取 item 高度（优先用缓存，没有则用预估）
  const getItemHeight = useCallback((index: number) => {
    return heightCache.current[index] ?? estimatedItemHeight
  }, [estimatedItemHeight])

  // 计算 item 位置（累加之前所有 item 的高度）
  const getItemPosition = useCallback((index: number) => {
    if (positionCache.current[index] !== undefined) {
      return positionCache.current[index]
    }

    let position = 0
    for (let i = 0; i < index; i++) {
      position += getItemHeight(i)
    }
    positionCache.current[index] = position
    return position
  }, [getItemHeight])

  // 计算总高度
  const totalHeight = useMemo(() => {
    let total = 0
    for (let i = 0; i < itemCount; i++) {
      total += getItemHeight(i)
    }
    return total
  }, [itemCount, getItemHeight])

  // 二分查找：找到 scrollTop 对应的 startIndex
  const findStartIndex = useCallback((scroll: number) => {
    let low = 0
    let high = itemCount - 1

    while (low <= high) {
      const mid = Math.floor((low + high) / 2)
      const position = getItemPosition(mid)
      const itemHeight = getItemHeight(mid)

      if (position + itemHeight < scroll) {
        low = mid + 1
      } else if (position > scroll) {
        high = mid - 1
      } else {
        return mid
      }
    }

    return Math.max(0, low - overscan)
  }, [itemCount, getItemPosition, getItemHeight, overscan])

  // 计算 startIndex 和 endIndex
  const { startIndex, endIndex } = useMemo(() => {
    const start = findStartIndex(scrollTop)
    let end = start
    let currentHeight = 0

    // 找到 endIndex：累加高度直到超过可视区域
    while (end < itemCount && currentHeight < height + overscan * estimatedItemHeight) {
      currentHeight += getItemHeight(end)
      end++
    }

    return {
      startIndex: Math.max(0, start - overscan),
      endIndex: Math.min(itemCount - 1, end + overscan),
    }
  }, [scrollTop, itemCount, height, estimatedItemHeight, findStartIndex, getItemHeight, overscan])

  // 处理滚动
  const handleScroll = (e: React.UIEvent<HTMLDivElement>) => {
    setScrollTop(e.currentTarget.scrollTop)
  }

  // 渲染后测量真实高度
  useEffect(() => {
    if (!containerRef.current) return

    const items = containerRef.current.querySelectorAll('[data-item-index]')
    items.forEach((item) => {
      const index = parseInt(item.getAttribute('data-item-index') || '', 10)
      const realHeight = item.getBoundingClientRect().height
      
      if (heightCache.current[index] !== realHeight) {
        heightCache.current[index] = realHeight
        positionCache.current = [] // 清空位置缓存，强制重新计算
      }
    })
  })

  // 渲染可视区域的元素
  const items = []
  for (let i = startIndex; i <= endIndex; i++) {
    items.push(
      <div
        key={i}
        data-item-index={i}
        style={{
          position: 'absolute',
          top: getItemPosition(i),
          width: '100%',
        }}
      >
        {children({ index: i, style: {} })}
      </div>
    )
  }

  return (
    <div
      ref={containerRef}
      style={{ height, overflow: 'auto', position: 'relative' }}
      onScroll={handleScroll}
    >
      <div style={{ height: totalHeight, position: 'relative' }}>
        {items}
      </div>
    </div>
  )
}
```

## 📚 三、面试准备

### 3.1 必背问题清单

#### Q1：为什么需要虚拟列表？

```
当列表数据量大时（通常超过1000条），直接渲染会导致：
1. DOM 节点过多，浏览器重绘重排开销大
2. 内存占用高，页面卡顿
3. 滚动性能差，用户体验差

虚拟列表只渲染可视区域的元素，DOM 节点从 O(n) 降为 O(1)，
保持稳定的渲染性能，不受数据量影响。
```

#### Q2：虚拟列表的核心原理是什么？

```
核心原理：
1. 计算可视区域能容纳多少个 item：visibleCount = height / itemHeight
2. 根据 scrollTop 计算 startIndex：startIndex = Math.floor(scrollTop / itemHeight)
3. 计算 endIndex：endIndex = startIndex + visibleCount
4. 只渲染 [startIndex - overscan, endIndex + overscan] 范围内的元素
5. 用一个高 div 撑开滚动区域，让滚动条正常显示

关键点：
- overscan 缓冲区：防止快速滚动时白屏
- absolute 定位：让每个 item 精确定位
- 滚动事件监听：动态更新渲染范围
```

#### Q3：定高 vs 不定高的区别？

```
定高：
- 每个 item 高度固定，可预先计算位置
- startIndex = Math.floor(scrollTop / itemHeight)
- 实现简单，性能好

不定高：
- 每个 item 高度不确定（如不同内容的卡片）
- 需要预估高度 + 渲染后实测
- 需要缓存已测量的高度
- 需要二分查找定位 startIndex
- 实现复杂，但更通用

不定高的核心策略：
1. 先用 estimatedHeight 预估
2. 渲染后用 getBoundingClientRect() 测量真实高度
3. 缓存高度，下次使用真实值
4. 位置需要动态重新计算
```

#### Q4：快速滚动时白屏怎么解决？

```
原因：滚动速度 > 渲染速度，新区域还没渲染出来

解决方案：
1. overscan 缓冲区：提前渲染可视区域外的几个 item
2. requestAnimationFrame 优化：使用 rAF 节流滚动事件
3. 占位符：在渲染完成前显示简单占位
4. 使用 Intersection Observer：更精确地检测元素可见性
```

#### Q5：react-window 的核心 API？

```
FixedSizeList：定高虚拟列表
- height: 可视区域高度
- itemCount: 数据总数
- itemHeight: 每个 item 高度
- rowComponent: 渲染组件
- rowProps: 传递给组件的 props

VariableSizeList：不定高虚拟列表
- estimatedItemHeight: 预估高度
- 需要实现 itemHeight 函数或使用 estimate

InfiniteLoader：无限滚动加载
- isItemLoaded: 判断是否已加载
- loadMoreItems: 加载更多数据的函数
- itemCount: 总数（可以是预估）
```
