### 问题 #1: onBlur事件处理函数写法错误

**首次记录**: 2026-03-10 16:30
**最近记录**: 2026-03-10 16:30
**记录次数**: 1

**原始问题**:
> 发现这个代码会使输入框直接显示value，但我想一开始只显示输入城市

**相关代码**:
```tsx
<input
    onBlur={() => handleBlur}
    value={cityName}
/>
```

**问题分析**:
箭头函数 `() => handleBlur` 只是返回了函数引用，并没有真正调用它。等于什么都没做。

**正确做法**:
```tsx
// 方式1: 箭头函数内调用
onBlur={() => handleBlur()}

// 方式2: 直接传递函数引用（如果函数签名匹配）
onBlur={handleBlur}
```

**学习要点**: ⭐ 箭头函数`() => func`只返回引用，不会执行。要调用必须`() => func()`

### 问题 #2: 受控组件缺少onChange事件

**首次记录**: 2026-03-10 16:30
**最近记录**: 2026-03-10 16:30
**记录次数**: 1

**原始问题**:
> 发现这个代码会使输入框直接显示value，但我想一开始只显示输入城市

**相关代码**:
```tsx
<input
    value={cityName}
    // 没有onChange事件
/>
```

**问题分析**:
input绑定了`value={cityName}`，但没有`onChange`事件来更新`cityName`。这会让输入框变成只读状态，用户无法输入任何内容。

**正确做法**:
```tsx
<input
    value={cityName}
    onChange={(e) => setCityName(e.target.value)}
/>
```

**学习要点**: ⭐ 受控组件使用value绑定就必须提供onChange更新方法，否则变成只读

### 问题 #3: 在查询前修改查询参数的逻辑错误

**最近记录**: 2026-03-10 18:00
**记录次数**: 2

**原始问题**:
> [2026-03-10 17:30] 为什么我的代码这么修改了还是无法正确的搜索？
> [2026-03-10 18:00] 为什么搜索两次才能输出正确结果？

**相关代码**:
```tsx
const handleKeyDown = (event: React.KeyboardEvent<HTMLInputElement>) => {
    if (event.key === 'Enter') {
        setCityName(getCityLocation?.poi?.[0]?.adm2 ?? '')  // 先修改参数
        refetchCityLocation()  // 再查询
    }
}
```

**问题分析**:
在触发查询**之前**就修改了查询参数（cityName），导致查询使用的是错误（空）的值。这是典型的"执行顺序错误"：应该是"用当前值查询 → 处理结果"，而不是"修改值 → 用错误值查询"。

**正确做法**:
```tsx
const handleKeyDown = async (event: React.KeyboardEvent<HTMLInputElement>) => {
    if (event.key === 'Enter' && cityName.trim()) {
        // 用当前用户输入的 cityName 查询，不要修改它
        const result = await refetchCityLocation()
        
        if (result.data?.poi?.[0]) {
            const poi = result.data.poi[0]
            onLocationChange(poi.lon, poi.lat)
        }
    }
}
```

**学习要点**: ⭐ 不要在查询前修改查询参数，也不要在 `setState` 后立即使用该状态。`setState` 是异步的，立即使用会得到旧值。解决方案：直接用变量本身，不要绕状态

### 问题 #1: placeholder不显示的原因

**首次记录**: 2026-03-10 16:30
**最近记录**: 2026-03-10 16:30
**记录次数**: 1

**原始问题**:
> 发现这个代码会使输入框直接显示value，但我想一开始只显示输入城市

**知识点**: React / HTML表单

**核心解答**:
placeholder只有在input的value为**空字符串**时才会显示。如果state初始值不是空字符串，placeholder永远不会显示。

**深入理解**:
这是HTML的标准行为，不是React特有的。placeholder是"占位文本"，意思是"这里本来是空的，所以我显示提示"。一旦有实际内容，提示就消失了。

**代码示例**:
```tsx
// ❌ 错误：初始值不是空字符串，placeholder不显示
const [cityName, setCityName] = useState('北京')

// ✅ 正确：初始值为空，placeholder会显示
const [cityName, setCityName] = useState('')
```

**学习要点**: ⭐ placeholder只在value为空字符串时显示，这是HTML标准机制

### 问题 #2: React Query enabled 选项的动态控制

**首次记录**: 2026-03-10 16:45
**最近记录**: 2026-03-10 16:45
**记录次数**: 1

**原始问题**:
> 我应该如何才能更改enabled的false值

**知识点**: React Query / 数据获取

**核心解答**:
`enabled` 是响应式的条件，可以传入状态变量或表达式。当条件从 `false` 变成 `true` 时，查询会自动执行。

**深入理解**:
`enabled` 不是设置一次就固定的值，它会在每次渲染时重新计算。你可以：
1. 直接传入条件表达式（如 `cityName.length > 0`）
2. 使用状态变量控制（如 `enabled: shouldSearch`）
3. 配合 `refetch()` 手动触发查询

**代码示例**:
```tsx
// 方式1：条件表达式（推荐）
useQuery({
    enabled: cityName.length > 0
})

// 方式2：状态变量
const [shouldSearch, setShouldSearch] = useState(false)
useQuery({
    enabled: shouldSearch
})

// 方式3：手动触发
const { refetch } = useQuery({ enabled: false })
const handleSearch = () => refetch()
```

**学习要点**: ⭐ `enabled` 是响应式的，条件变化会自动触发查询；或用 `refetch()` 手动触发

### 问题 #3: React状态更新的批处理机制

**首次记录**: 2026-03-10 17:00
**最近记录**: 2026-03-10 17:00
**记录次数**: 1

**原始问题**:
> 为什么我这样更改后 setSearchStatus(true) 然后又 setSearchStatus(false) 不起作用？

**知识点**: React / 状态管理

**核心解答**:
React 的状态更新是**异步批处理**的。连续调用多次 `setState` 时，React 会把它们合并成一次渲染，中间状态会丢失。

**深入理解**:
```tsx
setSearchStatus(true)   // 请求1
setSearchStatus(false)  // 请求2
// React 实际执行：直接变成 false，跳过了 true
```
这就像你要从1楼到3楼再到2楼，但电梯只在最终目的地停下——你不会在3楼停留。

**正确做法**:
```tsx
// ❌ 错误：中间状态会丢失
setStatus(true)
setStatus(false)

// ✅ 方式1：用 useEffect 监听状态变化
useEffect(() => {
    if (status) {
        // 执行操作
        setStatus(false)  // 完成后重置
    }
}, [status])

// ✅ 方式2：用异步操作
await doSomething()
setStatus(false)

// ✅ 方式3：换一种不依赖状态切换的方案
```

**学习要点**: ⭐ 连续的 `setState` 会被批处理合并，中间状态不会触发渲染。需要用 `useEffect` 监听或 `await` 等待

### 问题 #4: React Query refetch() 方法的使用

**首次记录**: 2026-03-10 17:15
**最近记录**: 2026-03-10 17:15
**记录次数**: 1

**原始问题**:
> refetch()的使用方法是什么？

**知识点**: React Query / 数据获取

**核心解答**:
`refetch()` 是手动触发查询重新执行的方法。配合 `enabled: false` 使用，可以完全手动控制查询时机，适合用户操作触发（点击、回车、下拉刷新）的场景。

**深入理解**:
```tsx
// 基本用法
const { data, refetch } = useQuery({
    queryKey: ['todos'],
    queryFn: fetchTodos,
    enabled: false  // 禁用自动查询
})

// 手动触发
const handleRefresh = async () => {
    const result = await refetch()  // 返回 Promise，可等待结果
    if (result.isSuccess) {
        console.log(result.data)
    }
}
```

**代码示例**:
```tsx
// 你的场景：按回车触发城市搜索
const { data: cityLocation, refetch } = useQuery({
    queryKey: ['city', cityName],
    queryFn: () => fetchCity(cityName),
    enabled: false
})

const handleKeyDown = async (e) => {
    if (e.key === 'Enter') {
        const result = await refetch()  // 触发查询并等待
        if (result.data) {
            onLocationChange(result.data.lon, result.data.lat)
        }
    }
}
```

**学习要点**: ⭐ `refetch()` 返回 Promise 可 await；配合 `enabled: false` 实现手动控制；用于用户操作触发的场景

### 问题 #5: String.trim() 方法的作用

**首次记录**: 2026-03-10 17:40
**最近记录**: 2026-03-10 17:40
**记录次数**: 1

**原始问题**:
> cityName.trim()有什么作用？

**知识点**: JavaScript / 字符串方法

**核心解答**:
`trim()` 用于去除字符串**两端**的空白字符（空格、制表符、换行符等），返回新字符串，不改变原字符串。在表单验证、搜索等场景，永远要先 trim() 再判断。

**深入理解**:
```tsx
const str = '  北京  '
console.log(str.trim())  // '北京'
console.log(str)         // '  北京  '（原字符串不变）

// 没有 trim() 的问题
const input = '   '  // 用户只输入了空格
if (input) {
    // '   ' 是 truthy，会执行
    // 但用空格查询，查不到结果
}

// 有 trim() 的正确做法
if (input.trim()) {
    // '   '.trim() → ''
    // '' 是 falsy，不会执行
}
```

**代码示例**:
```tsx
// 表单验证
const handleSubmit = () => {
    if (!username.trim()) {
        alert('请输入用户名')
        return
    }
    submit(username.trim())
}

// 搜索前清理
const handleSearch = () => {
    const keyword = searchInput.trim()
    if (keyword) {
        fetchResults(keyword)
    }
}
```

**相关方法**:
| 方法 | 作用 | 示例 |
|------|------|------|
| `trim()` | 去除两端空白 | `' a '.trim()` → `'a'` |
| `trimStart()` | 只去除开头空白 | `' a '.trimStart()` → `'a '` |
| `trimEnd()` | 只去除结尾空白 | `' a '.trimEnd()` → `' a'` |

**学习要点**: ⭐ `trim()` 用于清理用户输入，避免空格导致逻辑错误。表单验证、搜索等场景，永远先 trim() 再判断

### 问题 #6: 自定义搜索建议下拉列表的实现

**首次记录**: 2026-03-10 19:00
**最近记录**: 2026-03-10 19:00
**记录次数**: 1

**原始问题**:
> 如何在用户输入时显示地点提示列表？

**知识点**: React / UI组件

**核心解答**:
自定义下拉列表 = 输入框 + 实时搜索 + 条件渲染的列表 + 点击处理。核心是控制列表的显示/隐藏和点击外部关闭。

**深入理解**:
实现需要以下核心部分：
1. **状态管理**: 输入值、是否显示列表
2. **实时搜索**: useQuery + enabled 条件
3. **显示控制**: 条件渲染 `{show && data && <列表 />}`
4. **点击外部关闭**: useEffect + mousedown 事件监听
5. **选择处理**: 点击项目后更新状态、关闭列表

**代码示例**:
```tsx
// 1. 状态
const [input, setInput] = useState('')
const [showList, setShowList] = useState(false)
const inputRef = useRef(null)
const listRef = useRef(null)

// 2. 实时搜索
const { data } = useQuery({
    queryKey: ['suggestions', input],
    queryFn: () => fetchSuggestions(input),
    enabled: input.length >= 1  // 输入就触发
})

// 3. 点击外部关闭
useEffect(() => {
    const handleClick = (e) => {
        if (!listRef.current?.contains(e.target) &&
            !inputRef.current?.contains(e.target)) {
            setShowList(false)
        }
    }
    document.addEventListener('mousedown', handleClick)
    return () => document.removeEventListener('mousedown', handleClick)
}, [])

// 4. 选择处理
const handleSelect = (item) => {
    setInput(item.name)
    setShowList(false)
    onLocationChange(item.lon, item.lat)
}

// 5. 渲染
return (
    <div>
        <input
            ref={inputRef}
            value={input}
            onChange={e => { setInput(e.target.value); setShowList(true) }}
            onFocus={() => setShowList(true)}
        />
        {showList && data?.length > 0 && (
            <div ref={listRef} className="suggestions">
                {data.map(item => (
                    <div key={item.id} onClick={() => handleSelect(item)}>
                        {item.name}
                    </div>
                ))}
            </div>
        )}
    </div>
)
```

**学习要点**: ⭐ 自定义下拉列表五要素：状态管理、实时搜索、条件渲染、点击外部关闭、选择处理。关键是 useRef + useEffect 实现点击外部关闭
