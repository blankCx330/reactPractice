## ❌ 三、建议中的问题与遗漏

### 问题 1：优先级顺序可以优化

**原建议顺序**：
1. Tailwind → 2. Zustand → 3. React Query 深化 → ...

**更优顺序**：
1. **React Query 深化**（P0，零成本，面试必问）
2. **错误处理 + Loading**（P0，用户体验必备）
3. **Zustand + 收藏功能**（P1，状态管理能力）
4. **ECharts 可视化**（P1，视觉亮点）
5. **PWA 离线**（P2，30分钟完成）
6. **Vitest 测试**（P2，工程化意识）
7. **Tailwind 重构**（P3，时间充裕再做）

### 问题 2：遗漏的重要改进点

| 遗漏项 | 重要性 | 原因 |
|-------|-------|------|
| **错误处理 UI** | ⭐⭐⭐⭐⭐ | 当前无任何错误提示，API失败无感知 |
| **Loading 状态** | ⭐⭐⭐⭐ | 数据加载时无骨架屏/加载动画 |
| **搜索防抖** | ⭐⭐⭐⭐ | 当前每输入字符触发请求，浪费资源 |
| **API 封装层** | ⭐⭐⭐ | API 调用散落各组件，难以维护 |

### 问题 3：简历写法建议可更具体

**原建议**：
```
使用 React Query 处理接口数据缓存、状态与错误边界
```

**更优写法**：
```
基于 TanStack Query 实现数据请求与缓存策略（staleTime 5min，自动重试），
封装 useWeatherQuery 自定义 Hook，配合 Error Boundary 实现优雅降级，
搜索接口通过防抖优化减少 70% 无效请求
```
