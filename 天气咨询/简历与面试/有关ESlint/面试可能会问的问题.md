### 你的项目实际配置解析

面试时可能会被问到："你项目里 ESLint 配置了什么？"

#### 配置文件结构

```javascript
// eslint.config.js
export default defineConfig([
  globalIgnores(['dist']),           // 忽略 dist 目录
  {
    files: ['**/*.{ts,tsx}'],        // 检查所有 ts/tsx 文件
    extends: [
      js.configs.recommended,              // JS 基础规则
      tseslint.configs.recommended,        // TypeScript 规则
      reactHooks.configs.flat.recommended, // React Hooks 规则
      reactRefresh.configs.vite,           // Fast Refresh 兼容性
    ],
  },
])
```

#### 各插件作用

| 插件 | 检查什么 | 典型错误示例 |
|-----|---------|-------------|
| `js.configs.recommended` | JS 基础规范 | 未使用的变量、未定义的变量 |
| `tseslint.configs.recommended` | TypeScript 规范 | 隐式 any、类型不匹配 |
| `react-hooks` | Hooks 使用规范 | 条件中调用 Hook、依赖数组缺失 |
| `react-refresh` | Fast Refresh 兼容 | 导出非组件、副作用位置错误 |

### 模拟面试问答

#### Q1：介绍一下你们项目用的 ESLint

**参考回答**：

> 我们项目用的是 ESLint 9，配置了 TypeScript 和 React 的规则。主要检查三个方面：
> 1. TypeScript 类型问题，比如隐式 any
> 2. React Hooks 使用规范，比如 Hooks 不能在条件语句中调用
> 3. React Refresh 兼容性，确保 Fast Refresh 正常工作
>
> 我们用 flat config 格式配置，比传统的 `.eslintrc` 更灵活。开发时通过 `npm run lint` 检查，也会在 CI 中自动运行。

#### Q2：TypeScript 已经能检查类型错误了，为什么还要 ESLint？

**参考回答**：

> 它们职责不同：
> - TypeScript 检查**类型系统**的问题，比如变量类型不匹配、接口实现不完整
> - ESLint 检查**代码质量和风格**，比如未使用的变量、React Hooks 的正确使用
>
> 举个例子，TypeScript 不会告诉你"这个变量定义了但没用"，但 ESLint 会。TypeScript 不会检查 React Hooks 的依赖数组是否完整，但 `eslint-plugin-react-hooks` 会。
>
> 两者互补，一起使用能覆盖更全面的问题。

#### Q3：ESLint 规则有几种严重程度？

**参考回答**：

> 三种：
> - `off` 或 `0` - 关闭规则
> - `warn` 或 `1` - 警告，不会阻止构建
> - `error` 或 `2` - 错误，会导致构建失败
>
> 我们项目用的是 `recommended` 预设，已经预设了合理的严重程度。

#### Q4：如果某行代码确实需要违反规则，怎么办？

**参考回答**：

> 可以临时禁用规则：
> ```tsx
> // eslint-disable-next-line no-console
> console.log('debug info')
> ```
> 或者整个文件禁用：
> ```tsx
> /* eslint-disable no-console */
> ```
> 但应该少用，只在确实需要的时候用。

```

### 进阶学习建议

如果你想加深理解，可以：

1. **运行一次 lint** - 看看你的项目有没有警告/错误
   ```bash
   npm run lint
   ```

2. **故意写个错误代码** - 看 ESLint 怎么报错
   ```tsx
   // 尝试这些，看 ESLint 报什么错
   const unused = 123  // 未使用的变量
   if (true) { const [a, setA] = useState(0) }  // 条件中调用 Hook
   ```

3. **看看配置文件** - 理解每个插件的作用
   ```bash
   code eslint.config.js
   ```
