## 💻 三、核心代码实现

### 3.1 Zustand Store 实现

```typescript
// src/hooks/useFavoritesCityStore.tsx
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

type City = {
  id: string
  name: string
  adm2: string
  lon: string
  lat: string
}

type FavoritesCityStore = {
  list: City[]
  addCity: (city: City) => void
  removeCity: (id: string) => void
  isInList: (id: string) => boolean
}

export const useFavoritesCityStore = create<FavoritesCityStore>()(
  persist(
    (set, get) => ({
      list: [],

      // 添加收藏（带去重校验）
      addCity: (city: City) => {
        const exists = get().list.some(c => c.id === city.id)
        if (exists) return // 防止重复添加

        set(state => ({
          list: [...state.list, city],
        }))
      },

      // 移除收藏
      removeCity: (id: string) => {
        set(state => ({
          list: state.list.filter(c => c.id !== id),
        }))
      },

      // 判断是否已收藏
      isInList: (id: string) => {
        return get().list.some(c => c.id === id)
      },
    }),
    {
      name: 'favorites-city-list', // localStorage key
    }
  )
)
```

**技术要点**：

1. **persist 中间件**：自动持久化到 localStorage，刷新页面数据不丢失
2. **去重校验**：`addCity` 内部检查是否已存在，防止重复添加
3. **不可变更新**：使用展开运算符 `[...state.list, city]` 创建新数组

### 3.2 收藏按钮实现

```typescript
// src/TodaySWeather.tsx（收藏按钮相关代码）
import { useFavoritesCityStore } from './hooks/useFavoritesCityStore'
import { useLocationStore } from './hooks/useLocationStore'
import FavoriteIcon from './iconSVG/FavoriteIcon'

export default function TodaySWeather() {
  const location = useLocationStore(state => state.location)
  const { data: cityData } = useCityData(location.lon, location.lat)

  // 从 Store 获取方法和状态
  const { addCity, removeCity, isInList } = useFavoritesCityStore()

  // 生成城市唯一标识
  const currentCity = cityData?.location?.[0]
  const cityId = currentCity ? `${currentCity.adm2}-${currentCity.name}` : ''
  const isFavorited = isInList(cityId)

  // 收藏/取消收藏处理
  const handleFavoriteClick = () => {
    if (!currentCity) return

    if (isFavorited) {
      removeCity(cityId)
    } else {
      addCity({
        id: cityId,
        name: currentCity.name,
        adm2: currentCity.adm2,
        lon: currentCity.lon,
        lat: currentCity.lat,
      })
    }
  }

  return (
    <div className="today-weather">
      {/* 收藏按钮 */}
      <button
        onClick={handleFavoriteClick}
        className="favorite-btn"
        aria-label={isFavorited ? '取消收藏' : '添加收藏'}
      >
        <FavoriteIcon
          size={45}
          filled={isFavorited}  // 实心/空心状态
          className={isFavorited ? 'text-yellow-400' : 'text-gray-400'}
        />
      </button>
      {/* ... 其他内容 */}
    </div>
  )
}
```

**技术要点**：

1. **选择器优化**：只订阅需要的 `addCity`、`removeCity`、`isInList`，减少不必要的重渲染
2. **cityId 生成策略**：使用 `adm2-name` 组合确保唯一性
3. **图标状态联动**：`filled` prop 控制实心/空心，提供视觉反馈

### 3.3 收藏列表实现

```typescript
// src/FavoritesCityBtn.tsx
import { useState } from 'react'
import { useFavoritesCityStore } from './hooks/useFavoritesCityStore'
import { useLocationStore } from './hooks/useLocationStore'
import { FavoritesCityItem } from './FavoritesCityItem'

export default function FavoritesCityBtn() {
  const [isHide, setIsHide] = useState(true)

  // Store 数据
  const citys = useFavoritesCityStore(state => state.list)
  const removeCity = useFavoritesCityStore(state => state.removeCity)

  // 位置更新方法
  const setLocation = useLocationStore(state => state.setLocation)

  // 切换城市
  const handleCityClick = (city: City) => {
    setLocation({ lon: city.lon, lat: city.lat })
    setIsHide(true)  // 关闭下拉列表
  }

  // 渲染列表
  const renderList = () => {
    if (isHide) return null

    if (!citys.length) {
      return (
        <div className="favorites-empty">
          暂无收藏城市
        </div>
      )
    }

    return (
      <ul className="favorites-list">
        {citys.map(city => (
          <FavoritesCityItem
            key={city.id}
            city={city}
            onClick={() => handleCityClick(city)}
            onDelete={() => removeCity(city.id)}
          />
        ))}
      </ul>
    )
  }

  return (
    <div className="favorites-btn-container">
      <button
        onClick={() => setIsHide(!isHide)}
        className="favorites-trigger"
      >
        收藏列表
        <svg className="dropdown-icon">...</svg>
      </button>
      {renderList()}
    </div>
  )
}
```

### 3.4 列表项组件

```typescript
// src/FavoritesCityItem.tsx
interface Props {
  city: City
  onClick: () => void
  onDelete: () => void
}

export function FavoritesCityItem({ city, onClick, onDelete }: Props) {
  return (
    <li className="favorites-item">
      <span
        className="favorites-item-name"
        onClick={onClick}
      >
        {city.adm2} {city.name}
      </span>
      <button
        onClick={(e) => {
          e.stopPropagation()  // 阻止冒泡，避免触发 onClick
          onDelete()
        }}
        className="favorites-item-delete"
        aria-label="删除收藏"
      >
        ✕
      </button>
    </li>
  )
}
```

**技术要点**：

1. **事件冒泡处理**：`e.stopPropagation()` 防止删除按钮触发城市切换
2. **可访问性**：添加 `aria-label` 支持屏幕阅读器

## 🐛 四、常见问题与解决方案

### 4.1 问题：在组件渲染中直接调用 setState

**错误代码**：

```typescript
// ❌ 错误：会导致无限循环
const { data: cityData } = useCityData(lon, lat)

if (cityData) {
  setCityId(cityData.location[0].name) // 直接在渲染中调用 setState
}
```

**后果**：无限循环 → 页面崩溃 → React 报错 "Maximum update depth exceeded"

**正确做法**：

```typescript
// ✅ 正确：使用 useEffect 监听数据变化
const { data: cityData } = useCityData(lon, lat)

useEffect(() => {
  if (cityData?.location?.[0]?.name) {
    setCityId(cityData.location[0].name)
  }
}, [cityData])
```

**面试回答**：

> React 组件的渲染函数必须是纯函数，不能有副作用。`setState` 是副作用，必须在 `useEffect` 或事件处理函数中调用。直接在渲染中调用会导致无限循环。

### 4.2 问题：Zustand Store 命名规范

**错误命名**：

```typescript
// ❌ 不推荐：useNowCityId 看起来像普通 Hook
export const useNowCityId = (code: string) => { ... }
```

**正确命名**：

```typescript
// ✅ 推荐：use + 名词 + Store
export const useCityIdStore = create<CityIdState>((set) => ({ ... }))

// ✅ 工厂函数命名：create + 名词 + Store
export const createCityStore = (initial: InitialState) => create(...)
```

**命名规范表**：

| 类型       | 命名格式                  | 示例              |
| ---------- | ------------------------- | ----------------- |
| Store Hook | `use` + 名词 + `Store`    | `useCityIdStore`  |
| 工厂函数   | `create` + 名词 + `Store` | `createCityStore` |
| 派生 Hook  | `use` + 功能描述          | `useCurrentCity`  |

### 4.3 问题：判断数组是否为空

**错误写法**：

```typescript
// ❌ 错误：!citys[0] 判断的是第一个元素是否为假值
if (!citys[0]) { ... }
```

**问题场景**：

```typescript
const citys = [null, '北京'] // !citys[0] = true（误判为空）
const citys = ['', '上海'] // !citys[0] = true（误判为空）
```

**正确写法**：

```typescript
// ✅ 方式1：使用 .length
if (!citys.length) { ... }

// ✅ 方式2：显式判断
if (citys.length === 0) { ... }
```

## 💡 五、面试亮点总结

### 5.1 技术选型理由

**问：为什么用 Zustand 而不是 Redux？**

> 我选择 Zustand 主要有三个原因：
>
> 1. **简洁**：不需要 action、reducer、dispatcher 这些模板代码，一个函数定义 store
> 2. **性能**：内置选择器优化，只有用到的状态变化才触发重渲染
> 3. **开发体验**：TypeScript 支持开箱即用，持久化一个中间件搞定

### 5.2 数据流设计

**问：收藏功能的整体数据流是怎样的？**

> 数据流是单向的：
>
> 1. **用户操作** → 点击收藏按钮
> 2. **状态更新** → Zustand store 的 `addCity`/`removeCity`
> 3. **持久化** → `persist` 中间件自动写入 localStorage
> 4. **UI 响应** → 订阅 store 的组件自动重渲染
> 5. **列表展示** → 从 store 读取 `list` 渲染下拉菜单

### 5.3 性能优化

**问：如何避免不必要的重渲染？**

> 三个层面的优化：
>
> 1. **选择器优化**：只订阅需要的状态，比如 `useStore(state => state.list)` 而不是 `useStore()`
> 2. **条件渲染**：`isHide` 为 true 时不渲染列表，减少 DOM 操作
> 3. **事件冒泡控制**：删除按钮用 `stopPropagation()` 避免误触发城市切换

### 5.4 边界情况处理

**问：收藏功能有哪些边界情况需要处理？**

> 我处理了以下边界情况：
>
> 1. **重复收藏**：`addCity` 内部检查 `some(c => c.id === cityId)`，防止重复添加
> 2. **空列表**：`!citys.length` 时显示"暂无收藏城市"提示
> 3. **数据持久化**：使用 `persist` 中间件，刷新页面数据不丢失
> 4. **城市切换**：点击城市更新 `location`，React Query 自动触发数据重新请求
